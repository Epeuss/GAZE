Equipment Setup

Gaze signals were collected with an HTC VIVE Pro Eye headset in a Unity-based environment using SteamVR. Data were sampled at 100 Hz, and each image remained on screen for 1.0 seconds.

Acquisition Setting

Gaze signals were collected with an HTC VIVE Pro Eye headset in a Unity-based environment using SteamVR. The acquisition pipeline includes the headset, tracking units, and a workstation for scene rendering and data recording.

Data were sampled at 100 Hz, and each image remained on screen for 1.0 seconds during the experiment. Under this setting, each frame typically contains about 70 to 100 gaze points.

The collected gaze signals were then organized into frame-level coordinate files and corresponding visualization images for subsequent anomaly detection analysis and website-based browsing.

Data Structure

The platform manages two synchronized data sources for each frame: the gaze-overlay image and the coordinate txt file.

1

Dataset

Shanghaitech and UBnormal are stored separately.

2

Video

Each dataset contains multiple video folders.

3

Frame

Each video folder stores frame-level gaze-overlay images.

4

Coordinate TXT

Each frame is matched with its corresponding gaze coordinate file.
